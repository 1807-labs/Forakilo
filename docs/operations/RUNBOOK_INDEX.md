# Runbook Index

Purpose: list operational runbooks required before production.
Scope: MVP and controlled beta readiness.
Audience: operators, engineers, security reviewers, and risk owners.
Assumptions: detailed runbooks will be written with implementation.
Dependencies: [Incident Response](INCIDENT_RESPONSE_PLAN.md), [Alerting](ALERTING_AND_ESCALATION.md).
Unresolved decisions: runbook owners and tooling.

## Required Runbooks

- Authentication outage.
- Credential compromise.
- Provider API outage.
- Stale market data.
- Model drift or suspension.
- Kill switch activation.
- Unknown order outcome.
- Reconciliation mismatch.
- Database restore.
- Object storage restore.
- Secret rotation.
- Security vulnerability disclosure.
- Failed release rollback.
- Dependency vulnerability emergency.
