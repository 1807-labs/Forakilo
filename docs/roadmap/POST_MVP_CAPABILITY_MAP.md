# Post-MVP Capability Map

Purpose: list later capabilities without implying implementation.
Scope: post-MVP and future expansion ideas.
Audience: product owners, roadmap reviewers, and engineers.
Assumptions: later capabilities require new research, legal review, and ADRs.
Dependencies: [Delivery Roadmap](DELIVERY_ROADMAP.md), [Product Classification](../legal/PRODUCT_CLASSIFICATION_AND_REGULATORY_ASSUMPTIONS.md).
Unresolved decisions: prioritization and feasibility.

| Capability | Required before consideration |
| --- | --- |
| Additional venues | Provider terms, adapters, tests, and fallback strategy. |
| Additional assets | Legal classification, data contracts, and risk limits. |
| Portfolio optimization | Stronger model validation and risk governance. |
| Ensembles | Explainability, monitoring, and promotion controls. |
| Licensed sentiment | Contract, privacy, data rights, and model-risk review. |
| Mobile support | Security, session, and accessibility review. |
| Greater automation | Mature operations, proven paper/live evidence, and approval framework. |
